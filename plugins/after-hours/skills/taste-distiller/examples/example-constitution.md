# Example Constitution of Taste: Linear (the project management tool)

Distilled from observing the product surface, not from internal docs.

---

## What Gives It Its Feel

Dense but never crowded. Every screen shows a lot of information, but only one thing is visually dominant at a time. Motion is understated — things slide into place rather than bounce or flash. The product trusts you to find your way without hand-holding.

## Hidden Rules It Seems to Obey

- **One dominant action per view.** Secondary actions exist but are visually quieter.
- **Trust the user quickly.** No onboarding wizards, no tooltip storms. You start by doing, not by reading.
- **Keyboard-first, mouse-friendly.** Everything works with the keyboard, but nothing punishes the mouse user.
- **Motion is functional, not decorative.** Transitions communicate state change, not personality.
- **Information density stays high, visual noise stays low.** Achieved through restrained color, consistent type hierarchy, and generous but not wasteful spacing.
- **Error states are calm.** No red banners, no exclamation marks. Problems are stated plainly.

## What It Consistently Avoids

- Decorative illustration (no hero images, no mascots, no empty-state cartoons).
- Exclamation energy ("Welcome!" / "Great job!" / "You did it!").
- Over-explanation — it never tells you what a button does if the label is clear.
- Visual novelty for its own sake — every screen feels like it belongs to the same family.

## What Would Break the Effect

- Adding celebration animations to task completion.
- Using a friendly/casual tone in system messages.
- Adding a getting-started wizard that takes more than 30 seconds.
- Using different visual languages for different sections (e.g., making the settings page look like a different product).

## Portable Lessons

1. **Density is not clutter** if only one element is loud at a time.
2. **Trust accelerates adoption** — let people try before you explain.
3. **Consistent restraint feels more premium than occasional polish.**
4. **Calm error handling builds more trust than cheerful success states.**
