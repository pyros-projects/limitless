# Example Lorebook: Drift (a hypothetical note-taking tool)

---

## Beliefs

- Notes should feel like thinking out loud, not filing paperwork.
- The fastest path from thought to capture should be one keystroke.
- Structure should emerge from use, not be imposed before writing.
- Search is more important than organization.

## Refusals

- It refuses to become a project management tool.
- It refuses to require folders before you can write.
- It refuses to add collaboration features that slow down the solo path.
- It refuses to treat notes as finished documents.

## Recurring Metaphors

- Scratchpad, not notebook.
- Flow, not structure.
- Capture, not compose.

## Naming Rules

- Prefer verbs over nouns for actions: "capture", "find", "link" — not "note manager", "search engine", "connection hub."
- Features should sound like things you do, not things you install.
- Avoid words that imply hierarchy: no "folders", "categories", or "taxonomies." Use "tags" and "links."

## This Project Is / Is Not

| Is | Is Not |
|----|--------|
| A place to think | A place to publish |
| Fast and forgiving | Structured and permanent |
| Personal-first | Team-first |
| Messy on purpose | Messy by neglect |

## What Decisions This Makes Easier

- Should we add a templates feature? Only if it does not slow down blank-page capture.
- Should we add sharing? Only if it is opt-in per note and does not change the default writing experience.
- Should we redesign the sidebar? Only if search stays faster than browsing.
