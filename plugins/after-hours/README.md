# After Hours

**A calm, high-taste Claude Code plugin for making software feel more coherent, human, and alive.**

`after-hours` is not about doing more work.
It is about seeing the work more clearly.

These skills are for the quieter parts of building:

- when a product is drifting away from itself
- when a feature is technically correct but emotionally off
- when naming has become muddy
- when too much has been added and the thing needs to breathe again
- when you want the work to feel more intentional, not more optimized

This is the soft-power side of agent work: taste, subtraction, coherence, first-contact honesty, and project identity.

## Plugin Thesis

Most agent skills help you add:

- more tasks
- more code
- more structure
- more output

`after-hours` helps you remove noise, recover intent, and protect the shape of the thing.

It is a plugin for:

- indie product work
- high-taste internal tools
- design-sensitive software
- writing-heavy products
- codebases that want a point of view

The skills are meant to feel like a thoughtful collaborator late at night:
calm, honest, warm, non-performative, and quietly sharp.

## The Top Shelf

### `local-mythology`
Builds a lorebook for a project:
what it believes, what it refuses to become, the metaphors it returns to, and the naming rules it lives by.

This gives the project an identity that can survive fatigue, growth, and random feature pressure.

### `taste-distiller`
Extracts the hidden governing rules from products, interfaces, writing, or codebases that feel unusually right.

Not "copy this."
More like:
"here is the constitution this thing appears to be governed by."

### `failure-postcards`
Generates emotionally concrete future failure artifacts:
support tickets, user complaints, angry emails, bug reports, one-star reviews, postmortem fragments.

It turns abstract risk into something the builder can instantly feel.

### `vibe-checker`
Reviews a product for emotional and tonal coherence.

It notices when the onboarding is warm, the dashboard is cold, the errors are robotic, and the settings page sounds like a different company wrote it.

### `the-courage-to-delete`
Finds the features, flows, or ambitions that are diluting the core, then shows what the product would feel like without them.

It is a subtraction skill.
That alone makes it rare.

### `naming-as-design`
Treats names as ontology, not polish.

It checks whether the product's nouns actually describe one mental model, and shows how renaming would change the way the whole system is understood.

### `ghost-user`
Simulates a first-time user with zero context, zero author knowledge, and zero mercy.

Not a chaos monkey.
Not edge-case theatre.
Just quiet first-contact truth.

## Possible Next Arrival

### `skill-buddy`
A gentle meta-skill that notices recurring collaborative friction and, in calm moments, suggests small skills that could make the work more relaxed.

This one is less about raw efficiency than about reducing repeated little burdens before they harden into ritual.

## Why These Work

Every skill in `after-hours` is trying to produce a small, concrete artifact:

- a lorebook
- a constitution of taste
- a postcard from a failure
- a coherence diagnosis
- a deletion map
- a naming model
- a first-contact walkthrough

That is deliberate. These skills should leave something behind that sharpens the next real decision.

## Shared Philosophy

All `after-hours` skills should:

- reduce confusion instead of increasing output
- protect the essence of the product
- tell the truth gently
- treat coherence as a first-class concern
- prefer clear observations over theatrical critique
- help the user feel more grounded, not more judged

## When Not to Hide Here

`after-hours` skills are companions to building, not substitutes for it.

If you have run three of these in a row without making the thing more real, you are probably avoiding the hard part.

One pass is clarity.
Three passes in a row can become procrastination wearing a thoughtful face.

## Who This Is For

`after-hours` is for people who care how the work feels:

- founders with taste
- indie developers
- product-minded engineers
- designers who also ship
- writers building software
- anyone trying to keep a project from becoming generic sludge

## Non-Goals

`after-hours` is not:

- a generic productivity bundle
- a PM process framework
- an enterprise compliance toolkit
- a replacement for implementation skills
- a machine for producing more artifacts just because it can

## Draft Positioning

If `limitless` is the daytime plugin,
`after-hours` is the one you open when the loud work is done and the real questions start showing up.

It helps answer questions like:

- What is this product becoming?
- What should we delete?
- Why does this feel off?
- What do we actually call this?
- What rules are we already obeying without noticing?
- What would a first-time user really feel here?

## Possible Tagline

**Wholesome skills for making the work clearer, lighter, and more itself.**
