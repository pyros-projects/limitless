# Synthesis Patterns

## Purpose

Use these answer shapes to keep research output crisp.

Default rule:

- answer first
- support second
- extra depth only when it buys clarity

## Direct Answer

Use when the user asks a straightforward question.

Shape:

1. answer in 1-3 sentences
2. give the key evidence
3. include links if verification matters
4. offer 2-4 next directions if the pass was substantial

## Compare / Contrast

Use when the user asks about alternatives, frameworks, or tradeoffs.

Shape:

1. state the high-level judgment
2. explain the category split if the options are not doing the same job
3. compare on a small rubric
4. identify who should choose what
5. offer deeper next directions

## Change Over Time

Use when the user asks what changed, whether something is still true, or whether an old note drifted.

Shape:

1. say what changed
2. anchor with dates
3. distinguish stable core from changed surface
4. call out stale assumptions explicitly
5. offer next directions

## Truth Check

Use when the user asks if a claim is actually true.

Shape:

1. verdict: yes, no, mixed, or unclear
2. strongest supporting evidence
3. strongest limiting evidence
4. final confidence statement
5. offer ways to tighten confidence further

## Adoption Recommendation

Use when the user asks whether to use, buy, or adopt something.

Shape:

1. recommendation
2. why it fits or does not fit the user's actual situation
3. top risks and hidden costs
4. alternatives if relevant
5. next research directions

## Next Directions

At the end of substantial passes, propose 2-4 directions that are:

- concrete
- different from each other
- downstream of the evidence already found

Good directions:

- test the strongest counterclaim
- compare the best alternatives
- inspect the implementation details
- update the answer with a narrower time window

Weak directions:

- "learn more"
- "go deeper"
- paraphrases of the same branch
